# 통합 파이프라인 실시간 동기화 플랜 (v1.15.28)

## 1. 개요
- 가사-음원-이미지-영상-블로그로 이어지는 전체 공정을 하나의 유기적인 데이터 흐름으로 통합.
- 음원 분석 결과를 모든 탭(가사, 이미지, 렌더링)에 실시간으로 전파하고 DB에 영구 보존.

## 2. 핵심 작업 내역

### 2.1 [파일명 기반 가사 매칭] - `ImageTab.tsx` & `useMediaLogic.ts`
- 음원 업로드 시 파일명(`한글제목_영어제목`) 파싱.
- `sunoTracks` 리스트에서 제목이 일치하는 데이터 검색.
- 일치하는 데이터가 있을 경우 기존 가사를 즉시 로드.

### 2.2 [정밀 분석 및 데이터 역류] - `useMediaLogic.ts`
- 음원 분석 시 새로운 가사를 생성하지 않고, **기존 가사에 타임스탬프만 결합**.
- 분석 완료 즉시 `sunoTracks` 상태 업데이트 -> 모든 탭에 실시간 전파.
- 업데이트된 '타임스탬프 가사'를 Firestore에 즉시 저장하여 데이터 보존.

### 2.3 [렌더링 자막 동기화] - `VideoTab.tsx`
- 분석된 타임스탬프 데이터를 자막(`lyricsWithTimestamps`)으로 즉시 바인딩.
- 렌더링 시 해당 데이터를 서버에 전달하여 영상 싱크 완벽 구현.

### 2.4 [블로그 생성 최적화] - `BlogTab.tsx`
- 기존 생성된 이미지를 가로형으로 크롭(Crop)하여 사용.
- 이미지 위에 핵심 키워드/문구 오버레이 레이어 추가.

### 2.5 [버전 및 UI 동기화]
- `package.json`, `App.tsx` UI 버전을 `v1.15.28`로 업데이트.
- 가사 탭에서 불필요한 "원가사 입력", "영어곡 번역" 영역 삭제.

## 3. 원코드 - 수정코드 대조 (주요 로직)

**[파일명 파싱 및 가사 로드 로직 추가]**
```typescript
// useMediaLogic.ts 내 handleAudioUpload 보강
const fileName = file.name.split('.')[0]; // 한글제목_영어제목
const matchedTrack = sunoTracks.find(t => t.title === fileName || t.title === fileName.split('_')[0]);
if (matchedTrack) {
  setWorkflow(prev => ({ ...prev, results: { ...prev.results, lyrics: matchedTrack.lyrics } }));
}
```

**[분석 결과 전역 전파]**
```typescript
// 분석 완료 시점
const updatedTracks = sunoTracks.map(t => 
  t.title === currentTitle ? { ...t, lyricsWithTimestamps: analyzedData } : t
);
setSunoTracks(updatedTracks); // 전역 상태 업데이트 -> 모든 탭 동시 반영
```
