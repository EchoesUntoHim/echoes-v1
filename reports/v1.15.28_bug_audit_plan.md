## 📋 발견된 오류 목록

### 🔴 [오류 1] Firestore 권한 오류 - blogPosts & meditation_history (심각도: HIGH)

**증상:**
- 블로그 탭 진입 시: `Missing or insufficient permissions` → `blogPosts` 컬렉션 읽기 실패
- 1분 묵상 탭 진입 시: `Missing or insufficient permissions` → `meditation_history` 컬렉션 읽기 실패

**원인 분석:**
- `firestore.rules`에 `blogPosts` 컬렉션은 `allow list: if true` (공개)인데,
  코드에서는 `uid` 필드로 필터링하여 읽으려 해서 충돌 없이 정상이어야 하나,
  실제로는 `BlogTab.tsx`가 `where('uid', '==', user.uid)` 쿼리를 사용하여
  인증된 사용자여야 접근 가능 → 로그인 전 접근 시 오류 발생 (사용자 미로그인 상태).
- `meditation_history` 컬렉션은 `firestore.rules`에 **규칙 자체가 없음** (Default Deny).
  코드에서 `collection(db, 'meditation_history')`에 직접 쓰기/읽기하지만
  rules에는 이 컬렉션이 정의되어 있지 않음 → 모든 접근 차단됨.
- `generated_images` 컬렉션도 rules에 없음 → `useMediaLogic.ts`의 이미지 저장 시 실패.

**수정 계획:**
- `firestore.rules`에 다음 컬렉션 규칙 추가:
  1. `meditation_history` → 로그인된 사용자가 자신의 데이터만 읽기/쓰기/삭제 가능
  2. `generated_images` → 로그인된 사용자가 자신의 데이터만 읽기/쓰기 가능

### 🟡 [오류 2] React Key 중복 경고 (심각도: MEDIUM)

**증상:**
- 영상 렌더링, 영상 업로드, 설정 탭에서 `Encountered two children with the same key` 경고

**원인 분석:**
- 리스트 렌더링 시 `key` 값이 고유하지 않음 (index 또는 중복 id 사용)
- 정확한 위치는 `VideoTab.tsx`, `PublishTab.tsx`, `SettingsPage.tsx` 내부

**수정 계획:**
- 관련 파일 grep으로 `.map((item, idx) => ... key={idx})` 패턴 확인 후
  고유 key로 교체 (예: `key={item.id || idx}` 또는 `key={`${item.label}_${idx}`}`)

### 🟡 [오류 3] MeditationTab에 `addToRenderQueue`와 `isRendering` prop 미전달 (심각도: MEDIUM)

**증상:**
- 1분 묵상 탭의 "대량 렌더링" 버튼이 addToRenderQueue가 없으면 동작 안 함

**원인 분석:**
- `App.tsx` L842~L855의 `<MeditationTab>` 렌더링 시
  `addToRenderQueue`, `isRendering` props를 전달하지 않음
- 하지만 MeditationTab 내부(L663-L713) `handleBulkRender` 함수는 이 props를 사용함
- `useMediaLogic`에서 `addToRenderQueue`가 반환되고 있음 → 연결만 하면 됨

**수정 계획:**
- `App.tsx`의 MeditationTab 렌더링 시 `addToRenderQueue`, `isVideoRendering` props 추가

---

### 🟢 [오류 4] WebSocket HMR 연결 오류 (심각도: LOW)

**증상:**
- 콘솔에 `WebSocket connection to 'ws://localhost:3000/' failed` 반복

**원인 분석:**
- ccmpro 프로젝트의 dev 서버가 현재 구동 중이지 않음
- 브라우저가 InsightAI의 localhost:3000을 보고 있으나,
  ccmpro의 dev server가 아닌 빌드된 결과물이거나 다른 포트에서 실행 중일 수 있음

**수정 계획:**
- 사용자 확인 필요: ccmpro 전용 dev 서버(다른 포트)를 별도로 실행해야 함
- 코드 수정 불필요, 운영 환경 문제
