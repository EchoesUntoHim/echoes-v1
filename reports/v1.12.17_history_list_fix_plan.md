# [v1.12.17] 가사 생성 히스토리 누락 버그 수정 계획서

## 1. 개요
사용자님께서 제보하신 "가사를 생성했는데 히스토리 리스트에 나오지 않는 현상"을 해결하기 위한 계획입니다.

## 2. 문제 원인 (`App.tsx`)
가사 생성 로직(`generateLyrics`)을 확인한 결과, AI가 가사와 제목을 성공적으로 생성하여 화면의 입력창(`workflow.results`)에는 띄워주지만, 이것을 **히스토리 배열(`sunoTracks`)에 저장하는 코드가 아예 누락되어 있었습니다.**
이 때문에 새로 생성한 가사가 왼쪽 하단의 "가사 생성 히스토리 (Slim List)"에 나타나지 않고 증발해버리는 현상이 발생했습니다.

## 3. 해결 방안
`App.tsx`의 `generateLyrics` 함수 마지막 부분에 다음 로직을 추가합니다:
- 화면에 가사를 뿌려줄 뿐만 아니라, 생성된 제목, 가사, 영어 가사, Suno 프롬프트를 하나의 히스토리 객체로 묶어 `sunoTracks` 리스트 맨 앞에 추가(`setSunoTracks`)합니다.
- 이렇게 하면 버튼을 누르자마자 즉시 화면 하단 히스토리 리스트에 최신 생성본이 추가되며 클릭해서 언제든 다시 불러올 수 있게 됩니다.

## 4. 기대 효과
가사 생성 버튼을 누를 때마다 화면 하단의 히스토리 목록에 즉각적으로 제목과 날짜가 저장되어, 방금 만들었던 가사들을 잃어버리지 않고 안전하게 보관/불러오기 할 수 있습니다.
