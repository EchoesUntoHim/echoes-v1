# v1.5.6 UI 재배치 및 기능 수정 최종 작업 플랜

## 1. 개요
사용자의 최신 지시 사항 및 참고 이미지를 바탕으로 이미지 생성 탭의 UI를 재배치하고, 보고된 시스템 버그(자막, 노이즈)를 수정합니다. 모든 과정은 `rules/EXECUTION_RULES.md`를 준수하며, **'승인'** 후 실행합니다.

## 2. 상세 작업 항목

### [A] 이미지 생성 탭 UI 재배치 (src/components/ImageTab.tsx)
참고 이미지의 박스 위치를 기준으로 기능을 재배치합니다:
1. **이미지 업로드 영역 (빨간 네모)**:
   - 위치: 기존 한글/영어 제목 입력란 옆 빈 공간.
   - 구성: 업로드 대상(메인, 틱톡, 숏츠 1~5)을 선택할 수 있는 Dropdown/Toggle과 업로드 버튼 배치.
2. **음원 업로드 영역 (노란 네모)**:
   - 위치: 기존 이미지 업로드 기능이 있던 제목 입력란 하단 영역.
   - 기능: Suno 음원 업로드 버튼 배치.
3. **전체 이미지 생성 영역 (파란 네모)**:
   - 위치: 기존 우측 하단 또는 노란 네모 영역에 있던 생성 버튼을 이관.
   - 기능: AI 전체 이미지 생성 실행.

### [B] 숏츠 관리 기능 복구 및 정리
- **슬라이더 복구**: 우측 설정 패널에 '숏츠 이미지 수 설정(0~5)' 슬라이더 복구.
- **버튼 삭제**: "숏츠 보강생성" 버튼 삭제.
- **개별 생성 유지**: 선택한 숏츠 번호(체크박스) 기반 개별 생성 시스템 유지.

### [C] 시스템 버그 수정
- **자막 동기화 (VideoPlayer.tsx)**: 가사 원문(`lyrics`) 수정 시 인덱스 매핑을 통해 `timedLyrics` 데이터에 즉시 반영되도록 `useMemo` 로직 수정.
- **오디오 노이즈 (VideoPlayer.tsx)**: 숏츠 구간 변경 시 `audioRef.current.currentTime`을 즉시 강제 동기화하고, 미세한 볼륨 페이드 처리를 통해 팝 노이즈 방지.
