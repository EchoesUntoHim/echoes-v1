# [v1.12.23] v6 최종 정규화 및 미사용 음악 기능 삭제 플랜

본 계획서는 사용자님의 의도("가사/프롬프트는 유지, 미사용 중인 음악생성/편곡은 완전 삭제")를 완벽히 반영하여 프로젝트를 정규화하기 위한 최종 계획입니다.

## 1. 수정 핵심 원칙 (Core Rules)
- **가사 및 프롬프트 보존**: 현재 사용 중인 '가사 생성' 및 'Suno 프롬프트 생성' 관련 모든 UI와 로직은 **절대 수정하지 않고 유지**합니다.
- **미사용 기능 삭제**: 왼쪽 메뉴에 존재하지 않는 '음악 생성(Music Generator)', '편곡(Arrangement)', '음원 분리' 관련 모든 컴포넌트와 탭 로직을 삭제합니다.
- **엔진 설정 제거**: 설정 페이지에서 '음악 엔진' 선택 메뉴만 정밀하게 삭제합니다.
- **하드코딩 정리**: 버전 표기 및 API URL을 중앙 상수로 일원화합니다.

## 2. 세부 수정 항목

### [A] 미사용 음악 기능 완전 삭제
- **src/types.ts**: `Step` 타입에서 `arrangement`, `audio-separation` 제거. `MusicParams`에서 관련 속성 정리.
- **src/App.tsx**: 
    - `renderContent` 스위치 문에서 `arrangement`, `audio-separation` 탭 렌더링 로직 삭제.
    - 사이드바 메뉴는 현재 상태(가사, 음원리스트, 이미지, 영상 등)를 그대로 유지.
- **src/components/**: 
    - `ArrangementWorkspace.tsx` [DELETE]
    - `MusicGenerator.tsx` [DELETE]

### [B] 가사 및 프롬프트 기능 보존
- **src/App.tsx**: `generateLyrics`, `generatePromptOnly` 함수 및 Suno 프롬프트 생성 로직 100% 보존.
- **src/components/LyricsTab.tsx**: 프롬프트 입력창, 재생성 버튼 등 모든 UI 100% 보존.

### [C] 설정 및 하드코딩 정규화
- **src/constants.ts**: `DEFAULT_AI_ENGINE`, `DEFAULT_IMAGE_ENGINE`, `RENDER_API_URL` 상수 확립.
- **src/components/SettingsPage.tsx**: '음악 엔진' 드롭다운 UI 삭제.
- **전역**: 하드코딩된 모델명 문자열을 상태 변수(`aiEngine`, `imageEngine`)로 대체.

1. **가사/프롬프트 정상 작동 확인**: 기존처럼 가사와 프롬프트가 잘 생성되는지 확인.

3. **설정 페이지 확인**: 음악 엔진 설정이 깔끔하게 제거되었는지 확인.
