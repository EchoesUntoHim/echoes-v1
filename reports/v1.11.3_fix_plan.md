# v1.11.3 이미지 히스토리 접근성 개선 계획서

## 1. 분석 및 지침 반영
- **문제점**: 이미지 히스토리가 현재 작업 결과물(`workflow.results.images`)이 존재할 때만 화면에 노출됨. 이로 인해 과거 데이터를 불러와서 작업을 시작하려는 사용자 시나리오가 불가능함.
- **해결 방안**: 히스토리 영역을 조건부 렌더링 블록 밖으로 독립시켜, 이미지 생성 전에도 항상 접근 가능하도록 함.

## 2. 작업 목표
- 이미지 생성 탭 하단의 히스토리를 현재 생성 결과와 관계없이 상시 노출(데이터가 있을 경우).
- 사용자 경험(UX) 개선: 과거 작업물을 불러와서 현재 작업에 재사용하는 흐름 확보.

## 3. 세부 수정 계획
1. **[ImageTab.tsx]** 388번 라인에서 시작되는 `{workflow.results.images.length > 0 && (...)}` 블록의 닫는 중괄호 위치를 확인.
2. **[ImageTab.tsx]** 552번 라인의 히스토리 블록을 해당 조건문 밖(더 아래쪽)으로 이동.

## 4. 확인 및 기록
- 이미지 생성 전 상태에서 하단 히스토리가 정상적으로 보이는지 확인.
- `reports/v1.11.3_detailed_diff.md`에 변경 사항 기록.
