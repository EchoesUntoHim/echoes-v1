# [v1.12.9] UI/UX 시인성 극대화 및 메인 악기 펼침 메뉴 적용 계획서

## 1. 개요
- **디자인 가독성(Visibility) 개선**: 어두워서 잘 보이지 않는 추천 악기, 다중 선택 뱃지, 악기 텍스트 색상들을 밝고 직관적인 형태로 전면 수정합니다.
- **메인 악기 펼침 메뉴(Accordion) 도입**: 늘어난 악기 개수로 인해 화면이 길어지는 문제를 해결하기 위해, 메인 악기 선택 영역을 접고 펼칠 수 있는 구조로 변경합니다.
- **악기 추가**: '기타' 카테고리에 '아카펠라'를 추가합니다.

## 2. 상세 수정 계획

### 2.1 `src/components/LyricsTab.tsx` (UI 디자인 및 구조 수정)

#### A. 메인 악기 펼침 메뉴 (Expandable Menu) 구현
- `isInstrumentOpen` 상태(State)를 추가합니다.
- 메인 악기 영역의 헤더를 클릭 가능한 버튼("악기 목록 펼치기 ▼")으로 변경합니다.
- **선택된 악기 별도 표시**: 펼침 메뉴가 열려있든 닫혀있든 상관없이, **선택된 악기 목록은 메뉴 하단에 항상 눈에 띄게 표시**합니다.

#### B. 텍스트 및 색상 시인성(Visibility) 개선
1. **추천 악기 라벨**
   - 현행: 어두운 보라색 배경에 흰 글씨 (`bg-secondary/30 text-white`)
   - **변경**: 완전히 밝은 네온 색상(예: `bg-secondary text-black font-black px-2 py-1 rounded shadow-[0_0_15px_rgba(0,255,163,0.5)]`)으로 눈에 확 띄게 변경.
2. **"다중 선택 가능 (퓨전)" 뱃지**
   - **변경**: Primary/Secondary 그라데이션 및 네온 효과가 적용된 화려한 뱃지(`bg-gradient-to-r from-primary to-secondary text-black font-black px-3 py-1 rounded-full`)로 디자인 수정.
3. **악기 카테고리 제목 (건반악기 등)**
   - **변경**: 밝은 톤(`text-primary/90 font-bold`)으로 변경하여 구획 구분을 명확히 함.
4. **선택되지 않은 악기 버튼 텍스트**
   - **변경**: 기본 색상을 한 단계 밝은 회색(`text-gray-300`)으로 변경하여 식별력 극대화.

### 2.2 `src/constants.ts` (악기 추가)
- `INSTRUMENT_CATEGORIES`의 '기타' 항목에 **'아카펠라'** 문자열을 추가합니다.

### 2.3 버전 동기화
- `package.json` 및 `rules/APP_CODE_MAP.md`의 버전을 **v1.12.9**로 업데이트합니다.

## 3. 확인 절차
- 아카펠라 항목이 정상적으로 렌더링되고 선택되는지 확인합니다.
- 렌더링 시 펼침 메뉴가 정상적으로 작동하며, 하단에 선택된 악기가 명확히 표시되는지 확인합니다.
- 추천 악기 색상이 완전히 밝은 톤으로 적용되었는지 확인합니다.
