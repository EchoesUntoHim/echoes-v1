# App.tsx 코드 내비게이션 지도 구축 계획서

## 📝 개요
사용자님께서 15시간이나 고생하셨던 `App.tsx`의 뚱뚱한 코드 구조(3,600줄)로 인한 AI의 길 잃음 현상을 원천 차단하기 위한 플랜입니다. 코드를 물리적으로 분리하다가 겪은 서버 크래시 위험을 0%로 만들면서도, 기능별 논리적 분리(지도 작성)를 통해 빠른 탐색 및 수정을 가능하게 합니다.

## ⚠️ User Review Required
> [!TIP]
> 기존 코드는 **단 한 줄도 건드리지 않습니다.** 오직 AI가 `App.tsx` 내부 구조를 1초 만에 파악할 수 있도록 '내비게이션 문서'를 추가하고 '검색 강제 지침'을 주입하는 안전한 계획입니다. 검토 후 **"승인"**해 주시면 바로 실행하겠습니다.

## 🛠 세부 실행 계획

### 1. 코드 지도(Code Map) 파일 신규 작성
#### [NEW] `rules/APP_CODE_MAP.md`
- `App.tsx` 내부의 핵심 로직과 상태(State), 주요 AI 호출 함수들의 이름과 목적을 목차 형태로 구조화하여 기록합니다.
  - **상태 관리 영역**: `workflow`, `progress`, `apiKey` 등
  - **플랫폼 & 배포 영역**: `uploadToTikTok`, `uploadToBlogger`, `generatePlatformMetadata` 등
  - **AI 코어 로직**: `generateLyrics`, `generatePromptOnly`, `generateBlogPost` 등
  - **기타 헬퍼 함수**: `resetSubsequentSteps`, `addLog` 등

### 2. AI 강제 지침 (Strict Guidelines) 업데이트
#### [MODIFY] `rules/AI_STRICT_GUIDELINES.md`
- 기존 체크리스트 항목에 다음 지침을 강력하게 추가합니다.
- **[추가될 규칙 내용]**: 
  > "App.tsx 또는 거대한 코드를 수정할 일이 생기면, 절대 무지성으로 스크롤을 내리거나 전체 코드를 보려 하지 말 것. 반드시 사전에 `rules/APP_CODE_MAP.md`를 읽어 해당 기능이 포함된 '정확한 함수명'을 파악한 뒤, 검색(grep) 도구를 사용하여 단번에 해당 위치로 이동하여 수정할 것."

## ✅ 검증 방법
- 문서 생성이 완료된 직후, 가상의 시나리오(예: "블로그 메타데이터 생성 프롬프트 수정해봐")를 가정하여 AI가 얼마나 빠르게 위치를 찾아내는지 시연합니다.
