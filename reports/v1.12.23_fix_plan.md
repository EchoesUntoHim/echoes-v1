# v1.12.23 하드코딩 제거 및 버전 동기화 계획서

본 문서는 `rules/EXECUTION_RULES.md` 14번 규칙(하드코딩 금지) 준수와 프로젝트 전반의 버전 일관성을 확보하기 위한 수정 계획을 담고 있습니다.

## 1. 분석 결과 및 문제점
- **모델명 하드코딩**: `App.tsx` 내에서 AI 엔진 및 이미지 엔진 호출 시 특정 모델명이 폴백(Fallback) 값으로 직접 기술되어 있음.
- **버전 불일치**: `VideoPlayer.tsx` 내부 메타데이터가 `1.12.1`로 구버전 상태이며, `App.tsx` 일부 주석이 `v1.12.24`로 오기되어 있음.
- **URL 중복**: 로컬 렌더링 서버 주소가 여러 파일에 중복으로 하드코딩되어 있어 관리 효율성이 떨어짐.

## 2. 제안하는 수정 사항

### [MODIFY] src/constants.ts
- AI 및 이미지 모델의 기본값을 관리할 상수를 추가합니다.
- 로컬 렌더링 API URL을 상수로 정의합니다.
```typescript
export const DEFAULT_AI_ENGINE = 'gemini-3.1-flash-lite-preview';
export const DEFAULT_IMAGE_ENGINE = 'imagen-4.0-fast';
export const RENDER_API_URL = 'http://localhost:5000/render-shorts';
```

### [MODIFY] src/App.tsx
- 하드코딩된 모델명(`gemini-3.1-...`, `imagen-4.0-...`)을 위 상수로 대체합니다.
- `[v1.12.24]`라고 잘못 표기된 주석을 `[v1.12.23]`으로 수정하여 일관성을 확보합니다.

### [MODIFY] src/components/VideoPlayer.tsx & VideoTab.tsx
- `version: '1.12.1'`을 `version: '1.12.23'`으로 수정합니다.
- 하드코딩된 `http://localhost:5000/...` 주소를 `RENDER_API_URL` 상수로 대체합니다.

## 3. 검증 계획
- **정적 분석**: 수정 후 모든 파일에서 특정 모델명이 직접 노출되지 않는지 `grep`으로 재검토합니다.
- **기능 테스트**: 버전 정보가 화면 및 로그에 `1.12.23`으로 정상 노출되는지 확인합니다.
- **상세 보고**: 수정 완료 후 `reports/v1.12.23_detailed_diff.md`에 모든 변경 사항을 기록합니다.
