# 이미지 탭 레이아웃 복구 및 UI 최적화 계획서 (v1.15.24_restoration)

## 1. 개요
최근 작업 중 발생한 레이아웃 파손(업로드 섹션 증발) 및 드롭다운 메뉴 시인성 문제(흰 바탕에 흰 글씨)를 해결하고, 숏츠 개수 설정에 따른 동적 UI 연동을 완결하기 위한 복구 계획입니다.

## 2. 주요 수정 사항

### 가. 하단 업로드 바 레이아웃 복구 및 동적 연동
- **고정 항목**: `메인`, `틱톡` 업로드 버튼은 항상 표시.
- **동적 항목**: 사이드바에서 설정한 `숏츠 개수`만큼 `쇼츠 1`, `쇼츠 2`... 버튼이 즉시 생성되도록 연동.
- **기능**: 각 버튼 클릭 시 해당 위치의 이미지를 수동으로 교체할 수 있어야 함.

### 나. 이미지 생성 및 개별 생성 로직 정립
- **기본 생성**: '이미지 생성' 버튼 클릭 시 메인, 틱톡, 그리고 설정된 개수만큼의 숏츠 이미지를 한꺼번에 생성.
- **개별 생성**: 특정 숏츠 이미지만 다시 만들고 싶을 때 사이드바의 번호 선택 후 '개별 생성' 버튼 활용.

### 다. 다운로드 및 재생성 버튼 기능 강화
- **재생성**: 각 결과물 카드에 개별 재생성 버튼 배치.
- **다운로드 확인 로직**: 다운로드 버튼 클릭 시 "디자인된 타이틀을 포함하여 다운로드하시겠습니까?"를 묻는 팝업(또는 선택 모달) 출력.
    - '예': 캔버스에 그려진 디자인 포함본 다운로드.
    - '아니오': 원본 이미지만 다운로드.

### 라. 타이틀 컨트롤 패널(`VideoSettingsPanel`) 배치 및 연동
- **배치**: 메인, 틱톡 및 **각 개별 숏츠 이미지 하단**에 타이틀 설정 패널 노출.
- **연동**: 여기서 수정된 폰트, 크기, 위치, 효과 등은 `workflow.imageSettings`에 실시간 저장되어 **영상 렌더링(Video Tab) 페이지와 100% 동기화**됨.

### 마. 드롭다운(Select) 메뉴 가시성 해결
- **수정**: 모든 `select` 및 `option` 태그에 어두운 배경색(`bg-[#1A1F26]`)과 흰색 글자(`text-white`)를 강제 적용하여 시인성 확보.

### 바. 숏츠 개수 조절 바 UI 정밀화
- **수정**: 우측의 '추가 숏츠 이미지 수' 조절 바의 초록색 게이지가 선택한 숫자(1~5)에 맞춰 정확하게 채워지도록 시각적 완성도 높임.
