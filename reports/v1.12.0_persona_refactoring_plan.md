# [v1.12.0] AI 페르소나 외부화 및 리팩토링 실행 계획서

본 계획서는 `App.tsx` 내에 하드코딩된 대규모 AI 프롬프트를 독립된 `.txt` 파일로 분리하여 관리 편의성을 높이고, `rules/EXECUTION_RULES.md`의 규칙을 엄격히 준수하는 것을 목표로 합니다.

## 1. 버전 관리 정보 (Versioning)
- **현재 버전 (Current)**: `1.11.6`
- **다음 버전 (Next)**: `1.12.0` (Minor 업그레이드: 페르소나 외부화 및 기능 리팩토링)
- **버전업 시점**: 모든 수정 사항이 완벽히 반영되어 사용자가 "승인"한 최종 확인 직후.

- **로딩 확인**: `rules/EXECUTION_RULES.md` 및 `rules/AI_STRICT_GUIDELINES.md` 로딩 완료.

- **버전 관리**: 본 작업 완료 후 `1.11.6`에서 `1.12.0`으로 상향 예정.

## 2. 주요 작업 내용 (Core Tasks)

### A. 프롬프트 폴더 및 텍스트 파일 생성
`src/prompts/` 폴더를 생성하고 다음 파일들을 작성합니다.

#### 1. `src/prompts/Lyrics.txt`
- **구성**: 가사 작사 전용 페르소나 (`ccmPersona`, `popPersona`, `reformedMeditationPersona`)
- **특이사항**: 
    - 각 페르소나 상단에 `#` 주석으로 한글 번역 설명 추가.
    - **찬송가(Hymn)** 장르 선택 시 `Intro/Outro`를 제외하고 `Verse/Chorus`만 반복하는 구조 규칙을 영문 지침으로 명시.

#### 2. `src/prompts/Platform.txt`
- **구성**: 유튜브 및 틱톡 메타데이터 생성 페르소나
- **규칙 반영**: `EXECUTION_RULES.md`의 줄바꿈 규칙 강제 포함.
    - 유튜브 섹션 간 `\n\n` (Double newline)
    - 틱톡 캡션 내부 줄바꿈 및 해시태그 전 `\n\n\n\n` (4개 이상의 빈 행)
- **가변 처리**: `{{target}}` 등의 플레이스홀더를 사용하여 런타임에 동적 주입.

#### 3. `src/prompts/Blog.txt`
- **구성**: 네이버, 티스토리, 구글 블로그용 페르소나 및 장르 컨텍스트
- **가변 처리**: `{{userStyle}}`, `{{genreContext}}` 플레이스홀더 사용.
- **구성**: 영문 지침 + 한글 주석 설명.

### B. App.tsx 리팩토링
- **모듈 로딩**: Vite의 `?raw` import 기능을 사용하여 텍스트 파일을 문자열로 직접 임포트.
- **파싱 유틸리티**: 텍스트 파일에서 주석(`#`)을 제거하고 필요한 섹션을 추출하는 내부 함수 구현.
- **함수 업데이트**:
    - `generateLyrics`: 외부 텍스트의 페르소나를 주입하고 찬송가 전용 로직 적용.
    - `generatePlatformMetadata`: 외부 텍스트를 불러와 플레이스홀더를 실제 값으로 치환하여 프롬프트 구성.
    - `generateBlogPost`: 외부 텍스트를 기반으로 블로그 포스팅 프롬프트 구성.

### C. 추가 수정 사항 (발견된 오류 및 개선)
- **전통찬송가 판정 로직 수정**: `isHymn` 변수가 `'찬송가'`가 아닌 실제 상수값인 `'전통찬송가'`를 체크하도록 수정.
- **플랫폼별 맞춤 링크 적용**: 
    - **유튜브/틱톡 (`Platform.txt`)**: 리틀리 링크(`https://litt.ly/echoes`) 사용.
    - **블로그 포스팅 (`Blog.txt`)**: 유튜브 채널 링크(`https://www.youtube.com/@Echoes_Unto_Him`) 사용.
- **`App.tsx` 내 하드코딩 링크 제거**: `generateBlogPost` 등에서 예전 링크(`https://litt.ly/echoes`)가 하드코딩된 지침을 삭제하고, 외부 텍스트 파일의 지침만 따르도록 수정하여 링크 혼선을 방지.
- **가사 생성 로그 개선**: 전통찬송가 선택 시 적용되는 구조를 사용자에게 명확히 로그로 출력.
- **UI 버전 표시 업데이트**: `App.tsx`의 사이드바 로고 옆 및 `LandingPage.tsx`의 버전 표시를 `v1.12.0`으로 업데이트.
- **버전 규칙 명시화**: `rules/VERSIONING_RULES.md`에 "버전 변경 시 내부 코드(package.json 등)와 외부 UI(로고 옆 등)에 동시 표시해야 한다"는 규칙 추가.

## 4. 검증 계획 (Verification)
- **빌드 테스트**: `npm run dev` 시 타입 에러 및 임포트 에러 여부 확인.
- **링크 검증**:
    - 유튜브 메타데이터에 리틀리 링크가 포함되는지 확인.
    - 블로그 포스팅에 유튜브 채널 링크가 포함되는지 확인.
- **구조 테스트**: 전통찬송가 선택 시 가사에 인트로/브릿지/아웃트로가 포함되지 않는지 확인.
- **사용자 검토**: 모든 기능이 의도대로 작동하는지 사용자 확인 후 **백업** 진행.
