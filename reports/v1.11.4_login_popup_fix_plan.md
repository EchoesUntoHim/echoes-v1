# v1.11.4 구글 로그인 팝업 닫힘(무반응) 현상 해결 계획서

## 📝 현 상황 분석
사용자님께서 말씀하신 "팝업이 로딩되다가 사라지고 로그인 완료가 안 되는 현상"은, 과거 `v1.11.0` 당시 발생했던 **"LocalStorage 용량 초과(QuotaExceeded)로 인한 Firebase 인증 토큰 저장 실패 및 팝업 강제 종료"** 증상과 100% 일치합니다.
현재 앱에서 생성된 음악/가사 데이터(`suno_json_data` 등)가 브라우저 저장소의 한계치(약 5MB)를 꽉 채우고 있어서, 구글이 로그인 토큰을 발급해 줘도 브라우저가 "더 이상 저장할 공간이 없다"며 뱉어내고 팝업을 닫아버리는 것입니다.

## ⚠️ User Review Required
> [!IMPORTANT]
> 데이터를 강제로 지우면 사용자님의 소중한 작업물이 날아가므로, **"데이터는 보존하면서 저장 공간 부족 문제를 우회하고 에러를 명확히 보여주는 방향"**으로 수술을 진행하고자 합니다. 아래 계획을 확인하시고 **"승인"**해 주시면 바로 적용하겠습니다.

## 🛠 세부 수정 계획

### 1. 구글 로그인 에러 로그 완전 개방 (`src/App.tsx`)
- 기존에 `auth/popup-closed-by-user` 등의 에러를 숨기도록 처리된 로직을 수정하여, **로그인 실패 시 화면 좌측 하단 로그창(`addLog`)에 붉은 글씨로 "정확한 실패 원인(예: 저장소 부족, 네트워크 오류 등)"이 무조건 출력**되도록 변경합니다. 이를 통해 정확한 원인을 즉각 파악할 수 있습니다.

### 2. 저장 공간(LocalStorage) 다이어트 및 캐시 정리 버튼 추가
- 설정(Settings) 탭에 **[브라우저 캐시 비우기 (용량 확보)]** 버튼을 추가합니다. 
- 단, 사용자님의 귀중한 곡 데이터(`suno_json_data`)는 건드리지 않고, 비디오 렌더링 찌꺼기나 불필요한 임시 워크플로우 상태값(`echoesuntohim_workflow`)만 싹 비워서 안전하게 1~2MB의 공간을 확보하여 로그인이 뚫리도록 만듭니다.

### 3. Firebase 인증 유지 방식(Persistence) 최적화 검토 (`src/firebase.ts`)
- 용량이 꽉 찬 LocalStorage 대신, 용량이 널널한 `IndexedDB`나 `SessionStorage`를 우선 사용하도록 Firebase 초기화 설정을 명시적으로 조정하여 용량 부족으로 인한 튕김을 원천 방어합니다.

## ✅ 기대 효과
- 더 이상 팝업이 이유 없이 꺼지는 답답한 현상이 사라집니다.
- 용량이 부족하면 화면에 즉시 경고가 뜨며, [캐시 비우기] 버튼 한 번으로 곡 데이터를 유지한 채 로그인을 성공시킬 수 있습니다.
