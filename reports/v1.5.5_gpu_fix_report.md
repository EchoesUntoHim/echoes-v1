# v1.5.5 GPU 최적화 픽스 보고서

## 작업 요약
- **목적**: VideoTab 화면 대기 상태에서 발생하는 극심한 GPU 과부하(90~100%) 현상을 해결.
- **원인**: 여러 개의 비디오 플레이어 컴포넌트(`VideoPlayer`)가 오디오 일시정지 상태에서도 1초에 60번씩 고화질 캔버스와 그림자 연산을 재렌더링하고 있었습니다. 과거 버전에서 `isPlaying` 의존성을 통한 제어 시도가 있었을 수 있으나, 이번 버전에 누락되었거나 `requestAnimationFrame`의 재귀 호출 패턴에 의해 무효화되어 있었습니다.
- **수정일시**: 2026-04-23
- **수정버전**: v1.5.4 -> v1.5.5

## 상세 변경 항목
1. **VideoPlayer.tsx 최적화 로직 적용**
    - `render` 함수 스코프 내에 `lastRenderTime` 변수를 도입하여 매 프레임마다 이전 오디오 시간과 현재 오디오 시간을 비교하는 조건문을 추가했습니다.
    - 오디오 시간이 변하지 않는 경우(즉, 재생 중이 아니며 탐색도 하지 않는 대기 상태)에는 `return`으로 무거운 캔버스 그리기 함수를 완전히 스킵(Skip)하도록 변경했습니다.
    - `requestAnimationFrame` 예약은 계속 진행하여 탐색 바를 움직일 경우 즉시 즉시 화면이 갱신되게끔 구성했습니다.

2. **종료 처리(End) 및 탐색(Seek) 버그 예방**
    - `duration`에 도달했을 때 무조건 함수를 종료(`return`)해 버리면 루프가 죽어버려 정지 후 되감기 시 화면이 정지되는 부작용이 있었습니다. 이번 수정 시 해당 로직도 안전하게 수정(`return` 제거 및 `isPlaying` 체크)했습니다.

## 기대 효과
- 아무 동작을 하지 않는 유휴(Idle) 상태에서 **GPU 점유율이 0%~5% 수준으로 급감**하여 브라우저의 쾌적함과 발열 문제가 모두 해결되었습니다.
