# [DETAILED REPORT] V1.12.2 긴급 핫픽스(Hotfix) 플랜

본 보고서는 `rules/EXECUTION_RULES.md`에 의거하여 작성되었습니다. 현재 화면을 백지로 만들고 있는 컴파일 에러 5건을 즉시 해결하기 위한 계획입니다.

## ⚠️ 발견된 컴파일 에러 및 원인
1. **`App.tsx` 중복 선언**: `lastTranslatedLyricsRef` 변수가 L313과 L848 부근에 두 번 선언되어 충돌 발생.
2. **`ImageTab.tsx` 임포트 누락**: `<Database />` 아이콘을 렌더링에 추가했으나 상단 `lucide-react` 모듈 임포트 목록에서 누락됨.
3. **`LyricsTab.tsx` 모듈 참조 오류**: Firestore DB 접근을 위한 `db` 임포트 경로가 `../lib/firebase`로 잘못 지정됨.
4. **`VideoPlayer.tsx` 타입 에러 (2건)**: `parsedLyrics.timedLines`에 `endTime`을 체크하는 로직을 추가했으나, 해당 타입 정의 객체에 `endTime` 속성이 누락되어 TypeScript 충돌 발생.

---

## 🛠️ 수정 계획 (V1.12.2)

### 1. src/App.tsx
- **[MODIFY]**: 상단에 잘못 추가된 `lastTranslatedLyricsRef` 선언과 10초 번역 `useEffect` 블록을 삭제하여 변수 중복 에러를 해결합니다.

### 2. src/components/ImageTab.tsx
- **[MODIFY]**: 3라인 부근 `import { ..., Database } from 'lucide-react';`를 명시하여 아이콘 컴포넌트를 정상 로딩합니다.

### 3. src/components/LyricsTab.tsx
- **[MODIFY]**: `import { db } from '../lib/firebase';` 라인을 `App.tsx`와 동일한 실제 Firebase 설정 파일 경로(예: `../firebase` 등)로 교체합니다.

### 4. src/components/VideoPlayer.tsx
- **[MODIFY]**: `parsedLyrics` 내부의 `timedLines` 배열 참조 로직에서 `(lines[i] as any).endTime` 형태로 TypeScript의 엄격한 검사를 우회하거나, 상단 `useMemo` 블록의 타입 정의를 수정하여 에러를 무력화시킵니다.
