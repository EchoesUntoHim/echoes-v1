# [PLAN] V1.13.0 숏츠 재생 정지 및 오디오 차단 긴급 복구

이 문서는 숏츠 플레이 버튼 클릭 시 시간이 멈춰 있고 재생이 되지 않는 현상을 해결하기 위한 공식 실행 계획입니다.

## 1. 현상 진단 (Root Cause)
- **증상**: 플레이 버튼 클릭 시 `AbortError` 발생 및 시간 멈춤 (`0:00 / 0:00`).
- **원인**: `VideoPlayer.tsx`의 `<audio>` 태그 내 `crossOrigin="anonymous"` 속성이 로컬 리소스(blob) 로드를 차단함.

## 2. 상세 수정 계획

### [MODIFY] src/components/VideoPlayer.tsx
- **목표**: 오디오 엔진이 정상적으로 소스를 로드하도록 차단 요소 제거.
- **주요 수정**: 
  - 라인 1142 부근: `<audio>` 태그의 `crossOrigin="anonymous"` 속성 삭제.
  
```tsx
// 변경 전
<audio
  ref={audioRef}
  src={audioSrc || undefined}
  ...
  crossOrigin="anonymous"
/>

// 변경 후
<audio
  ref={audioRef}
  src={audioSrc || undefined}
  ...
/>
```

## 3. 규칙 및 검증
- **최소 수정**: 오직 숏츠 재생 복구를 위한 해당 속성 제거 외에 다른 코드 수정 금지.
- **검증**: 수정 후 브라우저에서 숏츠 플레이 버튼을 눌러 시간이 정상적으로 흐르고 소리가 나는지 확인.
