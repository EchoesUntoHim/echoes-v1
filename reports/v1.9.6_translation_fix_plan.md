# 번역 로직 최적화 계획서 (v1.9.0 - 중복 번역 충돌 해결)

## 1. 개요
음원 분석을 통해 확보된 고품질의 가사 및 번역 데이터가 기존의 자동 번역 `useEffect`에 의해 오염(Overwrite)되는 현상을 방지하기 위한 긴급 수정 계획입니다.

## 2. 상세 수정 계획 (`src/App.tsx`)
- **수정 위치**: 552행 부근의 가사 자동 번역 `useEffect`
- **수정 내용**: 
    - 가사(`lyrics`)와 번역본(`englishLyrics`)이 모두 이미 존재할 경우, 자동 번역 로직을 실행하지 않고 레퍼런스만 업데이트한 후 종료하도록 조건을 추가합니다.
```javascript
    // [수정 핵심 로직]
    // 만약 한글 가사와 영어 가사가 모두 이미 존재한다면 (음원 분석 완료 직후 등),
    // 중복 번역을 방지하기 위해 레퍼런스만 갱신하고 종료함.
    if (workflow.results.lyrics && workflow.results.englishLyrics) {
      lastTranslatedLyricsRef.current = workflow.results.lyrics;
      lastTranslateIsEnglishRef.current = workflow.params.isEnglishSong || false;
      return;
    }
```

## 3. 검증 계획
- 음원 분석 완료 후 터미널 로그에 "자동 번역 시작" 메시지가 중복으로 뜨지 않는지 확인합니다.
- UI의 한글 가사 편집창에 타임스탬프가 포함된 번역 가사가 정상 출력되는지 확인합니다.
