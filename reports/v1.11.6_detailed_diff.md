## 💻 코드 변경 상세 (Diff)

### 1. `src/App.tsx` (커스텀 OAuth 팝업 에러 감지 로직 추가 및 로고 버전 갱신)
**[변경 전]**
```typescript
const searchParams = new URLSearchParams(popupLocation.search);
const hashParams = new URLSearchParams(popupLocation.hash.substring(1));
const code = searchParams.get('code') || hashParams.get('code');

if (code) {
  clearInterval(checkToken);
  addLog("🔑 인증 코드를 획득했습니다. 토큰으로 교환합니다...");
```

**[변경 후]**
```typescript
const searchParams = new URLSearchParams(popupLocation.search);
const hashParams = new URLSearchParams(popupLocation.hash.substring(1));
const code = searchParams.get('code') || hashParams.get('code');
const error = searchParams.get('error') || hashParams.get('error');

if (error) {
  clearInterval(checkToken);
  addLog(`❌ 구글 승인 거절: ${error}`);
  authWindow?.close();
  return;
}

if (code) {
  clearInterval(checkToken);
  addLog("🔑 인증 코드를 획득했습니다. 토큰으로 교환합니다...");
```

- 좌측 상단 로고 버전: `v1.11.0` ➔ `v1.11.6` 으로 동기화

### 2. `src/firebase.ts` (구글 로그인 방식 팝업으로 완전 롤백)
**[변경 전 (1.11.x 리다이렉트 방식)]**
```typescript
export const signInWithGoogle = async () => {
  try {
    sessionStorage.setItem('oauth_intent', 'login');
    await signInWithRedirect(auth, googleProvider);
    return { user: null, accessToken: null };
```

**[변경 후 (1.10.0 팝업 방식 복구)]**
```typescript
export const signInWithGoogle = async () => {
  try {
    const result = await signInWithPopup(auth, googleProvider);
    const credential = GoogleAuthProvider.credentialFromResult(result);
    const accessToken = credential?.accessToken;
    return { user: result.user, accessToken };
```

### 3. `vite.config.ts` (보안 헤더 전면 삭제)
- `headers` 블록을 완전히 삭제하여 파이어베이스 및 커스텀 팝업 통신 차단을 원천 해제함.

## ✅ 결과 및 기대 효과
1. **버그 해결**: 구글 블로거 연동 시 권한 누락이나 API 미설정으로 에러가 발생해도 팝업창이 무한 대기하지 않고 즉각 에러 로그를 출력하며 닫힙니다.
2. **로그인 정상화**: 브라우저 헤더 정책 삭제로 구글 로그인이 과거 1.10.0 버전처럼 안정적으로 즉각 작동합니다.
3. **버전 규칙 완벽 준수**: 네이버/틱톡 등 미구현 기능이 아닌 순수 버그 픽스임이 확인되어 `v1.11.6(Patch)`으로 올바르게 패치되었습니다. (코드, 랜딩페이지, 앱 로고 모두 일치)
