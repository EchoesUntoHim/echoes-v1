In this update, we focused on UI restoration and cost optimization, ensuring the application remains stable after several refactoring phases.

### 1. UI Restoration: Lyrics History List
- **Problem**: The "Suno Tracks History" list in the Lyrics tab was not showing up because the required data (`sunoTracks`) was not being passed to the `LyricsTab` component.
- **Fix**: Updated `App.tsx` (L3364) to pass `sunoTracks` and `setSunoTracks` to `LyricsTab`.
- **Result**: Users can now view and select their previous lyric generations in the history list.

### 2. Cost Optimization: Smart Translation Guard
- **Problem**: Every time the page was refreshed, the app detected a change in the lyrics state and triggered an automatic translation (via Gemini API), leading to unnecessary token usage and costs.
- **Fix**: Implemented a "First Load Guard" in `App.tsx` (L845). The system now initializes the translation reference on the first render without calling the API.
- **Result**: Automatic translation only triggers when the user *actually* modifies the lyrics or settings, saving significant costs.
