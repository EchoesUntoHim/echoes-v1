# [PLAN] V1.13.0 클라우드 연동 에러 로그 상세화 및 통신 복구

이 문서는 V1.13.0 버전의 핵심인 클라우드 렌더링 서버 연동 실패 원인을 파악하고 해결하기 위한 공식 실행 계획입니다.

## 1. 목적
- 클라우드 렌더링 시 발생하는 `Failed to fetch` 에러의 원인을 시스템 로그에 투명하게 노출.
- 사용자 지시에 따른 "시스템 로그에 에러 내용 있는 그대로 출력" 원칙 준수.

## 2. 상세 수정 계획

### [MODIFY] src/components/VideoPlayer.tsx
- **목표**: 서버 요청 실패 시 브라우저가 제공하는 원본 에러 메시지를 시스템 로그로 전송.
- **주요 수정**:
  - `fetch().catch()` 블록에서 `addLog`를 호출하여 실제 `err.message` 및 시도한 `videoRenderApiUrl` 출력.
  - `mode: 'cors'`, `credentials: 'omit'`, `Accept` 헤더 유지 및 강화.

### [MODIFY] src/App.tsx
- **목표**: 상위 컴포넌트에서도 렌더링 중단 시 에러의 전체 내용을 시스템 로그에 기록.
- **주요 수정**:
  - `handleDownloadAll`의 `catch` 블록에서 `error.stack` 또는 `error.message`를 가감 없이 로그에 출력.

## 3. 규칙 및 준수 사항
- **최소 수정 원칙**: 지시된 에러 로그 강화 외의 타 로직 및 UI 수정 절대 금지.
- **보고 체계**: 작업 완료 후 `V1.13.0_cloud_rendering_connectivity_fix_detailed_report.md` 작성.
