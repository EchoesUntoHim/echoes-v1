# 통합 파이프라인 초정밀 시스템 명세서 (v1.15.28)

본 문서는 Echoes Unto Him 프로젝트의 데이터 아키텍처, 비즈니스 로직, 기술적 세부 규격을 정의한 최종 마스터 플랜입니다.

## 1. 데이터 아키텍처 및 연동 (Data Architecture)
### 1.1 Firestore 데이터 스키마 (`sunoTracks` 컬렉션)
- `id`: 문서 고유 ID (제목 기반 또는 랜덤 ID)
- `title`: 곡 제목 (파일명 파싱 결과)
- `lyrics`: 가사 원문 ([벌스], [코러스] 태그 포함)
- `lyricsWithTimestamps`: 타임스탬프가 포함된 가사 데이터 (JSON Array)
- `generatedImages`: 생성 및 업로드된 이미지 URL 배열
- `audioUrl`: Firebase Storage에 업로드된 음원 주소
- `userId`: 사용자 고유 식별자 (auth.uid)
- `created_at`: 생성 및 업데이트 일시 (serverTimestamp)

### 1.2 파일명 파싱 정규식
- 업로드 파일: `한글제목_영어제목.mp3`
- 매칭 로직: `file.name.split('_')[0]`을 통해 1차 검색, 실패 시 전체 파일명으로 2차 검색.

## 2. 지능형 음원 분석 파이프라인 (Audio Analysis)
### 2.1 조건부 분석 프로세스
1. **히스토리 매칭**: 업로드 파일명과 DB 내 `title` 필드를 대조.
2. **분석 모드 결정**:
    - **매칭 시**: 기존 `lyrics` 로드 -> AI 분석 서버에 전달 -> **가사는 유지하고 타임스탬프만 매핑**.
    - **미매칭 시**: 음원 전체 분석 -> **가사 추출(STT) + 타임스탬프 동시 생성**.
3. **결과 전파**: 분석 완료 즉시 `sunoTracks` 업데이트 및 모든 활성 탭(가사, 이미지, 영상)에 전역 State 실시간 브로드캐스팅.

## 3. 미디어 가공 및 자동화 규격 (Media Processing)
### 3.1 블로그 이미지 최적화
- **Center Crop**: 9:16 세로 이미지를 16:9 가로 규격으로 변환 시, 중앙 영역을 기준으로 자동 크롭.
- **Contextual Overlay**: **블로그 생성 프로세스의 맥락**에서 도출된 중요 포인트 및 문구를 이미지 위에 시각적으로 오버레이.

### 3.2 전역 렌더링 큐(Rendering Queue) 및 업로드
- **Sequential Processing**: 영상 렌더링 서버(FFmpeg)의 과부하 방지를 위해 모든 제작 요청(영상 + 묵상)을 단일 대기열에서 순차적으로 처리.
- **플랫폼 업로드 전략**:
    - **유튜브**: API를 통한 자동 업로드 및 키워드 최적화 수행.
    - **틱톡**: API 연동 보류로 인해 **사용자가 직접 수동으로 업로드** 진행.
- **Cleanup Strategy**: 유튜브 자동 업로드 및 틱톡 수동 업로드 완료를 확인한 후, `downloads/` 폴더의 원본 파일 삭제 명령 수행.

## 4. UI/UX 및 버전 정합성
- **히스토리 리스트 공유**: 가사 탭과 이미지 탭 하단에 `sunoTracks` 기반의 통합 리스트 배치.
- **로컬 캐싱**: Firestore 로딩 전 `localStorage`에 저장된 데이터를 먼저 보여주어 사용자 체감 속도 최적화.
- **버전 동기화**: `package.json`, `App.tsx` 사이드바 로고 옆 버전 표기를 `v1.15.28`로 일치시킴.
- **UI 정리**: `LyricsTab.tsx`에서 실사용하지 않는 '원가사 입력', '영어 번역 설정' 영역 삭제.
