# v1.11.0 기능 연동 및 UI 간소화 작업 계획서

## 1. 분석 및 실패 원인 파악
- **UI 수정 실패**: 사이드바 디자인 변경 시 리액트 주석(`{/* */}`) 문법과 닫는 태그(`</div>`) 불균형으로 인해 '백지 현상' 발생.
- **로그인 실패**: `LocalStorage` 용량 초과로 인해 Firebase 인증 상태 저장이 차단되어 팝업이 강제로 닫히는 현상 발견.
- **기능 미비**: 블로그 연동은 토큰 획득까지만 구현되었으며, 실제 업로드 버튼과의 연결이 누락됨.

## 2. 작업 목표
- 사이드바 상단 브랜드명과 버전을 박스 없이 한 줄로 표시 (버전은 숫자만 표시).
- 블로그 탭의 업로드 버튼을 실제 API 서비스와 연동.
- 저장 공간 부족으로 인한 로그인 오류를 방지하기 위해 자동 정리 로직 추가.

## 3. 세부 수정 계획

### [Component 1] App.tsx
- **버전 표시**: `v1.11.0 PREMIUM` 및 박스 디자인을 모두 삭제하고, 브랜드명 옆에 `1.11.0` 숫자만 표시되도록 한 줄 디자인으로 변경.
- **용량 관리**: 앱 시작 시 `localStorage`의 `echoesuntohim_sunoTracks` 크기를 체크하여 3MB 초과 시 오래된 기록을 자동 삭제하는 안전장치 추가.
- **토큰 전달**: `BlogTab` 컴포넌트에 `bloggerAccessToken`을 props로 전달.

### [Component 2] BlogTab.tsx
- **인터페이스**: `BlogTabProps`에 `accessToken` 속성 추가.
- **업로드 연동**: `[연동된 플랫폼에 업로드]` 버튼 클릭 시 `uploadToBlogger` 함수를 호출하여 실제 포스팅 수행.

## 4. 검증 계획
- 코드 수정 전후로 `npm run dev` 터미널 로그를 실시간 모니터링하여 문법 에러 유무 확인.
- 브라우저 콘솔을 통해 `LocalStorage` 용량 확보 및 로그인 팝업 정상 작동 여부 재확인.
- 블로그 업로드 성공 후 생성된 URL로 실제 게시물 확인.
