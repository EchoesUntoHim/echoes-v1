# [v1.12.12] 통합 오류 수정 계획서 (유튜브 권한 버그 & 가사 악기 이름 삽입 버그)

## 1. 개요
사용자님께서 보고해주신 두 가지 심각한 오류(구글 로그인 후 유튜브 업로드 실패 문제, 악기 이름이 가사에 직접 삽입되는 문제)를 동시에 해결하기 위한 **v1.12.12 통합 패치 계획**입니다.

## 2. 상세 수정 계획

### 2.1 가사 텍스트 내 악기 이름 삽입 버그 (프롬프트 수정)
- **원인**: 메인 악기 다중 선택(퓨전) 기능이 고도화되면서, AI가 `Main Instrument` 파라미터를 '가사의 소재'로 착각하여 "피아노가 울려퍼지고..." 와 같이 악기 이름을 가사에 직접 적어버리는 현상입니다. (페르소나 자체가 망가진 것은 아니며, AI의 파라미터 과적합 문제입니다.)
- **해결 방안 (`App.tsx` 프롬프트 영역 수정)**:
  - 가사 생성 지침(Lyrics section)에 아래의 강력한 제어 규칙을 추가합니다.
  - `- **NO INSTRUMENT NAMES IN LYRICS (CRITICAL)**: The 'Main Instrument' parameter is STRICTLY for the 'Suno AI Prompt' to design the musical arrangement. NEVER mention the literal instrument names (e.g., piano, guitar, drum) inside the generated lyrics. The lyrics must remain a pure, natural poem.`
  - 이를 통해 AI가 악기를 가사 내용이 아닌 '편곡용 데이터'로만 인식하도록 강제합니다.

### 2.2 유튜브 업로드 구글 로그인 권한 버그
- **원인**: 이전 보고서에서 말씀드린 대로, 유튜브 업로드 전용 로그인 창에서 유튜브 스코프(권한)가 없는 일반 로그인 함수(`signInWithGoogle`)를 호출하고 있었습니다.
- **해결 방안 (`App.tsx` 수정)**:
  - `App.tsx` 1393번째 줄의 코드를 `signInForYouTube()` 함수 호출로 변경하여 정상적으로 유튜브 업로드 권한을 획득하도록 수정합니다.

### 2.3 버전 동기화
- `package.json` 및 `rules/APP_CODE_MAP.md`의 버전을 **v1.12.12**로 업데이트하고 수정 내역을 기록합니다.

## 3. 확인 절차
- 악기를 다중 선택(예: 피아노, 해금)한 뒤 생성된 가사에 "피아노", "해금" 등의 단어가 들어가지 않는지 확인합니다.
- 유튜브 연동이 해제된 상태에서 유튜브 업로드를 시도할 때, 정상적으로 구글 로그인 창이 뜨고 업로드가 진행되는지 확인합니다. (단, 로컬 `localhost` 환경이어야 합니다.)
