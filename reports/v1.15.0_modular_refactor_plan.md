# v1.15.0 App.tsx 모듈화 리팩토링 계획서

## 1. 개요
- **버전**: v1.14.1 -> v1.15.0
- **목적**: 3,700라인이 넘는 거대한 `App.tsx` 파일을 기능별 커스텀 훅으로 분리하여 코드 복잡도를 낮추고 유지보수성을 향상시킴.
- **핵심 원칙**: 
  - **기존 UI/UX 100% 유지** (사용자가 느끼는 변화 없음)
  - **기존 기능 100% 유지** (로직의 결과물 동일)
  - **안정성 우선** (롤백 가능하도록 단계별 진행)

## 2. 상세 작업 내용

### 2.1 커스텀 훅 통합 및 로직 이식
현재 `src/hooks`에 이미 생성된 훅들을 최신 `App.tsx` 로직에 맞춰 업데이트하고 연동합니다.

1. **`useAuthLogic`**: 
   - Firebase 인증 상태 (`user`), 로그인/로그아웃 로직 관리.
2. **`useHistoryLogic`**: 
   - Suno 트랙 히스토리, 로컬 스토리지 마이그레이션, 클라우드 동기화 로직.
3. **`useLyricsLogic`**: 
   - 가사 생성, 분석, 번역, 제목 재생성 로직.
4. **`useMediaLogic`**: 
   - 음원 업로드/분석, 이미지 업로드, 비디오 렌더링, 다운로드 로직.
5. **`usePlatformLogic`**: 
   - 유튜브/틱톡/블로거 API 토큰 관리 및 연동 모달 상태 관리.
6. **`useContentLogic`**: 
   - 플랫폼 메타데이터 생성, 블로그 포스팅 생성, 이미지 생성 총괄.

### 2.2 App.tsx 구조 축소
- 모든 `useState`, `useEffect`, 핸들러 함수들을 해당 훅으로 이동.
- `App.tsx` 상단에서 훅을 호출하여 필요한 상태와 함수만 가져옴.
- JSX 부분(UI)은 그대로 유지하여 렌더링 구조 보존.

## 3. 작업 단계 (Surgery Step - 단계별 즉시 검증)

1. **[Step 1: 준비]** 현재 `App.tsx`를 `App_backup_v1.14.1_stable.tsx`로 복사하여 안전 장치 확보.
2. **[Step 2: 인증/히스토리 분리]** `useAuthLogic`, `useHistoryLogic`을 먼저 연동. (로그인 및 데이터 로딩 기능 검증)
3. **[Step 3: 가사/콘텐츠 로직 분리]** `useLyricsLogic`, `useContentLogic` 연동. (AI 생성 기능 검증)
4. **[Step 4: 미디어/플랫폼 로직 분리]** `useMediaLogic`, `usePlatformLogic` 연동. (업로드 및 렌더링 기능 검증)
5. **[Step 5: App.tsx 슬림화]** UI 코드만 남기고 나머지 잔여 로직 정리.
6. **[Step 6: 최종 검증]** 모든 탭 간의 이동 및 데이터 보존 상태 전수 점검.

## 4. 위험 관리 및 원칙
- **상태 동기화**: 모든 훅은 `workflow` 상태를 공유하며, 상호 간의 의존성을 최소화하기 위해 훅이 다른 훅을 호출하지 않고 `App.tsx`에서 데이터를 중재하도록 설계.
- **클로저 문제 방지**: `useCallback`과 `useEffect`의 의존성 배열을 철저히 관리하여 상태가 꼬이는 현상 방지.
- **중단 시 복구**: 매 단계 완료 시마다 팅김에 대비하여 현재 진행 상황을 `reports/`에 기록.
