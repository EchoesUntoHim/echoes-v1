# v1.10.0 Release Report

## 1. 개요
v1.10.0 버전은 영상 렌더링 엔진의 안정화와 런타임 오류 해결을 목표로 하는 마이너 업데이트입니다. 주요 변경 사항으로 FFmpeg 엔진 버전을 안정화 버전(v0.11)으로 교체하고, 앱 전반의 undefined 참조 오류를 해결하였습니다.

## 2. 주요 수정 내용 및 코드 비교 (Rule 10 준수)

### 2.1 FFmpeg 렌더링 엔진 안정화 (VideoPlayer.tsx)
- **변경 사유**: v0.12 버전의 엄격한 보안 요구사항(COOP/COEP)으로 인한 로딩 중단 및 무한 대기 현상 해결.
- **조치**: v0.11.0 안정 버전으로 다운그레이드 및 로컬 엔진 파일(/ffmpeg/) 설치.

#### [수정 전 (v0.12 style)]
```typescript
import { FFmpeg } from '@ffmpeg/ffmpeg';
import { fetchFile, toBlobURL } from '@ffmpeg/util';

const loadFFmpeg = async (addLog) => {
  ffmpeg = new FFmpeg();
  await ffmpeg.load({
    coreURL: await toBlobURL(`/ffmpeg/ffmpeg-core.js`, 'text/javascript'),
    wasmURL: await toBlobURL(`/ffmpeg/ffmpeg-core.wasm`, 'application/wasm'),
  });
};
```

#### [수정 후 (v1.10.0 / v0.11 style)]
```typescript
import { createFFmpeg, fetchFile } from '@ffmpeg/ffmpeg';

const loadFFmpeg = async (addLog) => {
  ffmpeg = createFFmpeg({
    log: true,
    corePath: '/ffmpeg/ffmpeg-core.js',
    logger: ({ message }) => { if (addLog) addLog(`[FFmpeg] ${message}`); }
  });
  await ffmpeg.load();
};
```

### 2.2 이미지 생성 페이지 런타임 에러 수정 (ImageTab.tsx)
- **변경 사유**: `imageParams` 객체가 초기화되지 않은 상태에서 `artStyle` 속성에 접근할 때 발생하는 크래시 해결.

#### [수정 전]
```typescript
<select value={workflow.imageParams.artStyle} ... />
```

#### [수정 후]
```typescript
<select value={workflow.imageParams?.artStyle || '실사 사진 (Photorealistic)'} ... />
```
