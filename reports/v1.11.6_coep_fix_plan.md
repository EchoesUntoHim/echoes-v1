# 🚨 최종 원인 분석: 파이어베이스 투명 창(iframe) 차단

사용자님의 완벽한 제보 덕분에 미스터리가 100% 풀렸습니다!
구글로 넘어갔다 돌아왔는데도 앱이 모른 척(결과값 Null) 하는 이유는, 브라우저의 영상 렌더링 보안 모드(`require-corp`) 때문이었습니다.

## 📝 원인 상세 규명
1. 파이어베이스는 구글에서 돌아올 때, 화면에 보이지 않는 **투명한 창(iframe)**을 띄워서 로그인 결과(토큰)를 몰래 전달받습니다.
2. 하지만 1.11.3 버전에서 도입된 프리미엄 영상(FFmpeg) 보안 모드는 외부에서 들어오는 모든 투명 창의 접근을 "해킹"으로 간주하고 **원천 차단(`require-corp`)**해 버립니다.
3. 그래서 구글은 토큰을 던져주는데, 파이어베이스 투명 창이 차단되어 앱이 토큰을 받지 못하고 멍때리게 된 것입니다!

## 🛠 최종 수술 계획: 보안망에 파이어베이스 전용 '개구멍' 뚫기
보안 모드를 끄면 영상 기능이 죽고, 켜면 로그인이 죽습니다.
둘 다 살리기 위해서는 `vite.config.ts` (브라우저 서버 설정)에 걸린 철통 보안망을 살짝 헐겁게(`credentialless`) 만들어 주어야 합니다.

### 수정 내용 (`vite.config.ts`)
```typescript
      headers: {
        'Cross-Origin-Opener-Policy': 'same-origin',
        // 기존: 'Cross-Origin-Embedder-Policy': 'require-corp' (모두 차단)
        // 변경: 파이어베이스 투명창은 허용하되 영상 렌더링(SharedArrayBuffer) 권한은 유지
        'Cross-Origin-Embedder-Policy': 'credentialless',
      },
```

## ✅ 기대 효과
이 설정 한 줄을 바꾸면:
- FFmpeg 영상 렌더링 엔진은 정상 작동합니다.
- 파이어베이스가 띄우는 로그인 인증 창(iframe)이 차단되지 않아, 구글에서 돌아왔을 때 앱이 정상적으로 토큰을 건네받습니다.
- 1시간 넘게 괴롭히던 "구글 로그인 무반응/튕김" 버그가 마침내 소멸합니다.
