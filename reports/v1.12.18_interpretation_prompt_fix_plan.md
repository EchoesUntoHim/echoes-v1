# [v1.12.18] 곡 해석(Interpretation) 기계적 반복 수정 계획서

## 1. 개요
사용자님의 피드백을 반영하여, AI가 '곡 해석(Interpretation)'을 작성할 때 매번 선택된 악기(예: "이 곡은 피아노와 어쿠스틱 기타를 바탕으로...")를 앵무새처럼 반복하는 현상을 수정하는 계획입니다.

## 2. 문제 원인 (`App.tsx`)
현재 AI는 사용자가 선택한 '메인 악기' 파라미터를 그대로 읽어들여, 곡 해석의 첫 줄에 기계적으로 끼워 넣는 뻔한 패턴을 보이고 있습니다. 가사에는 악기 이름을 빼라고 지시해 두었으나(v1.12.12), 곡 해석에는 이런 제한을 두지 않았기 때문입니다.

## 3. 프롬프트 수정 방안
`App.tsx`의 JSON 포맷 지시사항 중 `"interpretation"` 항목을 다음과 같이 강화하겠습니다:
- **[기계적 반복 금지]**: 사용자가 선택한 메인 악기나 장르 이름을 앵무새처럼 그대로 나열하지 말 것.
- **[다양성 및 깊이 확보]**: 악기를 직접 언급하기보다는 전체적인 '사운드의 공간감', '감정의 고조(빌드업)', '신학적/시적 의미'를 깊이 있게 해설하여 매번 새롭고 다채로운 해석이 나오도록 강제합니다.

## 4. 기대 효과
이제 곡 해석을 읽을 때마다 뻔한 악기 설명이 아닌, 마치 전문 평론가나 프로듀서가 곡의 영적 메시지와 전체적인 사운드 스케이프를 깊이 있게 설명해 주는 듯한 세련된 해설을 볼 수 있습니다.
