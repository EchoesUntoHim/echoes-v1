# v1.11.3 구글 로그인 및 연동 오류 수정 계획서

## 📝 현 상황 분석
사용자님의 정확한 진단대로, 1.10.0에서는 멀쩡했던 구글 로그인이 1.11.x에서 '연동 기능'이 추가되면서 완전히 고장 났습니다.
원인은 **'단순 앱 접속용 구글 로그인'과 '유튜브/블로그 업로드용 구글 연동'이 하나의 인증 객체(Provider)를 돌려쓰고 있기 때문**입니다. 연동을 위해 억지로 권한을 섞다 보니, 정작 기본 로그인조차 안 되고 업로드 권한도 못 받아오는 총체적 난국이 발생했습니다.

## ⚠️ User Review Required
> [!IMPORTANT]
> 이 계획은 15시간의 고생이 담긴 `refactoring_plan.md`(안전하게 보존됨 확인!)을 본격적으로 시작하기 전에, **가장 시급한 심장부(로그인/연동) 버그를 고치는 수술**입니다. 아래의 완벽한 분리 계획을 확인하시고 **"승인"**해 주시면 즉시 코드를 수정하겠습니다.

## 🛠 세부 수정 계획

### 1. 두 개의 독립된 구글 인증 체계 구축 (`src/firebase.ts`)
기존에 하나뿐이던 `googleProvider`를 역할에 맞게 **두 개로 완벽히 분리**합니다.
- **[수정]** `googleProvider` (기본): 이메일/프로필만 요구하는 아주 가벼운 '앱 접속 전용' 제공자.
- **[신규]** `youtubeProvider` (연동용): `.addScope('...youtube.upload')` 및 `.addScope('...blogger')` 코드를 추가하여, 유튜브 업로드와 블로거 쓰기 권한만을 전문적으로 받아오는 '연동 전용' 제공자 신설.

### 2. UI 호출 로직 완전 분리 (`src/App.tsx`)
- **앱 기본 로그인 버튼 (3265줄 근처)**: 무조건 기본 `googleProvider`만 사용하여 빠르고 안전하게 앱 로그인 수행.
- **플랫폼 연동 토글 버튼 (315줄 근처 - `handlePlatformLoginConfirm`)**: 
  - 사용자가 개인 API 키를 넣지 않았을 경우, 억지로 기본 로그인을 호출하던 코드를 삭제.
  - 대신 신설된 `youtubeProvider`를 통해 팝업을 띄우도록 수정하여, **로그인과 연동이 서로 간섭하지 않도록 원천 차단.**

## ✅ 기대 효과
1. **구글 로그인 즉시 정상화**: 더 이상 연동 로직이 기본 로그인을 방해하지 않아 1.10.0 때처럼 원활하게 로그인됩니다.
2. **연동 100% 성공 보장**: 유튜브/블로그 전용 권한(`youtube.upload`)을 구글 서버에 정확하게 요청하므로, 생성된 영상이 문제없이 유튜브에 자동 업로드됩니다.
