### 🔴 [수정 1] 음원 업로드 시 불필요한 자동 번역 발동 (HIGH)

**증상:**
- 음원 업로드 시 "번역 중..."이 표시됨
- 영어 가사 음원의 경우, 이미 분석 단계에서 한/영 가사가 모두 세팅되었음에도 10초 후 자동 번역이 재실행됨

**원인 분석:**
- `App.tsx` L376-383: `workflow.results.lyrics` → `videoLyrics` 싱크 시
  `lastTranslatedLyricsRef`가 업데이트되지 않음
- L482-503: `videoLyrics`가 변경되면 무조건 10초 후 `translateLyrics()` 발동
- 음원 분석(`analyzeAudioComprehensively`)이 이미 양국 가사를 모두 세팅했음에도, ref가 갱신되지 않아 중복 번역 발생

**수정 계획:**
- `App.tsx` L376-383의 싱크 useEffect에서, `workflow.results.englishLyrics`도 존재하는 경우
  `lastTranslatedLyricsRef.current = workflow.results.lyrics`로 업데이트
- 이렇게 하면 음원 분석 후 자동 번역이 발동하지 않음 (이미 번역 완료이므로)

**수정 파일:** `App.tsx` (L376-383)

### 🔴 [수정 2] 묵상 이미지 도서관: 이미지가 보이다 사라짐 + 기존 이미지 불러오기 실패 (HIGH)

**증상:**
- 이미지 생성 후 도서관에 잠깐 보이다 사라짐
- 기존에 생성된 수십 장의 이미지가 불러와지지 않음

**원인 분석:**
- `MeditationTab.tsx` L169의 Firestore 쿼리:
  `where('userId', '==', user.uid)` + `orderBy('created_at', 'desc')`
  → **복합 인덱스(Composite Index)**가 필요하나 생성되지 않았을 가능성 높음
- 복합 인덱스가 없으면 쿼리가 실패하고 데이터가 빈 배열로 돌아옴
- 또한 `serverTimestamp()`는 로컬에서 null로 시작 → 정렬 불안정

**수정 계획:**
- `orderBy`를 제거하고 클라이언트 사이드에서 정렬 (인덱스 의존성 제거)
- 이미 통합 라이브러리 (L143-152)에서 클라이언트 정렬을 하고 있으므로
  historyQ에서 `orderBy`만 제거하면 즉시 해결됨
- 또는 Firestore 콘솔에서 인덱스를 수동 생성 (배포 불필요 대안)

**수정 파일:** `MeditationTab.tsx` (L169)

### 🟡 [수정 4] 영상 플레이 시 진입 효과(Entry Animation) 미적용 - 깜빡임 (MEDIUM)

**증상:**
- 영상 플레이 시 타이틀/이미지에 진입 효과(fade, slide 등)가 적용되지 않고 깜빡임만 발생

**원인 분석:**
- `VideoPlayer.tsx` L829-833의 렌더 루프에서 `currentAudioTime !== lastRenderTime` 체크
  → 재생 전(isPlaying=false)에는 `audio.currentTime`이 변하지 않으므로
  초기 프레임이 그려지지만, 재생 시작 순간 `currentTime`이 미세하게 변하면서
  화면이 다시 그려지며 **깜빡임** 발생
- L508: `isPrePlay = !isPlaying && elapsed <= 0.1` → 재생 시작 직후 isPrePlay가 false로 전환되며
  `titleOpacity`가 순간적으로 0→1 점프 (fade-in이 0.5초인데 elapsed가 이미 0 이상)
- **핵심 원인**: 재생 시작 시 `startTime` 기준으로 elapsed가 바로 양수가 되어
  fade-in 구간이 사실상 스킵됨

**수정 계획:**
- 재생 시작 시점(playStartedAt)을 기록하고, elapsed 계산 시 실제 재생 시작 후 경과 시간을 사용
- `togglePlay` 함수에서 재생 시작 시 `playStartedTimeRef.current = Date.now()` 기록
- `drawFrame` 내에서 진입 효과의 elapsed를 `(Date.now() - playStartedTimeRef) / 1000`으로 계산
  → 이렇게 하면 매 재생마다 진입 효과가 처음부터 정확히 실행됨

**수정 파일:** `VideoPlayer.tsx` (L343-620 drawFrame, L866-878 togglePlay)
