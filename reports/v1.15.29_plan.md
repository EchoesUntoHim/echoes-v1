# 수술 플랜 v1.15.29 (Revision 1): 가사 우선순위 및 자동 매칭 파이프라인

## 1. 개요
이미지 생성 탭에서 음원 업로드 시, 파일명을 분석하여 기존 히스토리(sunoTracks)에서 동일한 제목의 작업을 자동으로 찾아냅니다. 일치하는 데이터가 있을 경우 기존 생성 가사를 최우선으로 사용하며, AI는 오직 타임스탬프 동기화 작업만 수행합니다.

## 2. 주요 수정 내역

### 2.1 useMediaLogic.ts (자동 매칭 로직)
- **파일명 파싱 강화**: `한글_영어제목` 형식을 정확히 추출.
- **히스토리 검색**: `sunoTracks`에서 해당 제목과 일치하는 트랙을 탐색.
- **가사 전달**: 일치하는 트랙 발견 시 해당 트랙의 `lyrics`와 `englishLyrics`를 `analyzeAudioComprehensively`의 `referenceLyrics`로 자동 전달.

### 2.2 useLyricsLogic.ts (분석 가이드 강화)
- **우선순위 지침**: `referenceLyrics`가 제공될 경우, 추출된 가사보다 이 가사가 "무조건 정답"임을 명시하여 텍스트 수정을 엄격히 금지.
- **타임스탬프 오버레이**: 기존 가사의 구조를 유지하며 정확한 시간 정보만 입히는 작업에 집중하도록 프롬프트 조정.

### 2.3 데이터 연동 (App.tsx / 전 탭 공유)
- 분석 완료 시 `results`와 `params`를 동시에 업데이트하여 가사 탭 및 영상 탭의 모든 입력칸이 즉시 동기화되도록 함.

### 2.4 UI 최적화 (ImageTab.tsx)
- 업로드 버튼 높이 `38px` 축소 및 실시간 파일명 노출 유지.

## 3. 규칙 준수 사항
- 수정 전후 상태를 **규칙 3-3**에 따라 엄격히 검증.
- 작업 완료 후 **규칙 4**에 따라 버전업 및 코드맵 기록.
