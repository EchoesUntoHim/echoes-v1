# 사용자 지시 전 확인 사항 (Execution Rules)

1. **지시사항 이외의 코드는 절대 건들지 말 것.**
2. **지시 명령을 하면, 이 파일을 먼저 로딩해서 확인 후 진행할 것.**
3. **모든 지시사항에는 반드시 플랜으로 보고하고 검토 후 승인이 되면 진행할 것**
3-1 ** 모든 명령에서 수정사항은 반드시 플랜으로 보고하고 대기할 것**
3-2 **'승인' 이 외에 단어에는 절대 코드 수정하지 말 것. ('수정', '실행', '고쳐' 등 사용 금지)**
3-3. **코드 수정 후 반드시 확인 절차를 거쳐서 에러 없는 상태를 만들  것.**
4. **코드가 수정된 작업을 하면, 반드시 버젼규칙에 따라 버젼업 표시할 것.**
5. **버젼업이 되면 작업내용을 상세하게(원코드 - 수정코드 전체기록할 것) 기록할 것.**
6. **작업 중 오류가 발견되면 보고하고 대기할 것.**
7. **모든 보고는 한글로 한다.**
8. **수정 사항이 완벽하게 동작함이 확인된 직후에 전체 파일 백업을 수행한다. (압축 파일로 저장, 이름은 현재 버전으로 하며 backups 폴더에 보관한다. 최신 5개 파일만 남기고 삭제한다.) 완벽히 동작함은 사용자가 판단한다.**
9. **전체백업파일 저장시 같은 버전치 있을 경우, 이전 저장파일 살제후 저장**
10. **모든 보고는 파일 형태로 출력하고 보관한다.**
11. **모든 생성 파일은 각 해당 폴더에 저장한다.**
12. **깃허브 업로드 시 백업 폴더(backups/)는 항상 제외한다. (.gitignore에 등록하여 관리)**

## 버전 관리 규칙 (Reference)
- **x.0.0 (Major)**: 전체 구조 또는 디자인 개편.
- **0.x.0 (Minor)**: 페이지 내 기능 변화 또는 단일 메뉴 수정.
- **0.0.x (Patch)**: 단순 오류 수정 또는 작은 UI 트윅.
