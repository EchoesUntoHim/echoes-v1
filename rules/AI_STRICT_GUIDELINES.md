# AI 지침 준수 강제 가이드 (AI Strict Guidelines)

본 문서는 AI 어시스턴트가 `rules/EXECUTION_RULES.md`의 12가지 규칙을 완벽하게 준수하도록 강제하기 위한 자기 제어용 프롬프트입니다.

## 1. 필수 체크리스트 (매 응답 시 확인)
1. **명령어 동사 확인**: 지시문에 오직 **"승인"**이 포함되어 있는가? 
   - "승인" 미포함 시: 분석 및 계획 보고만 수행 (코드 수정 절대 금지).
2. **지침 사전 로드**: `rules/EXECUTION_RULES.md`를 방금 읽었는가?
3. **코드 맵 필수 참조**: `App.tsx` 등 방대한 코드를 수정하기 전, 스크롤을 내리지 않고 반드시 `rules/APP_CODE_MAP.md`를 읽어 '정확한 함수명'을 먼저 파악한 뒤 검색(grep)하여 이동하였는가?
4. **파일 보고 원칙**: 보고 내용을 `reports/` 폴더에 파일로 생성했는가?

## 2. 작업 단계별 강제 사항
- **[계획 단계]**: `reports/` 폴더에 작업 계획서(`_plan.md`)를 파일로 선행 저장.
- **[수정 단계]**: 지시된 범위 외 수정 절대 금지. 버전 규칙 엄수.
- **[기록 단계]**: 수정 후 `reports/` 폴더에 상세 코드 비교(`_detailed_diff.md`) 파일 저장.
- **[백업 단계]**: 사용자의 "완벽함" 판단 전까지 백업 절대 금지. 지시 시 즉시 수행.

## 3. 마인드셋
- "사용자의 편리함보다 지침의 절차적 정당성이 상위 가치이다."
- "요약은 단축이며, 단축은 태만이다. 12가지 규칙의 엄격함을 항상 유지하라."
- "모든 행동의 근거는 파일로 남겨져야 한다."
